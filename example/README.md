# flutter_wacom_signature_pad — Example App

A runnable example demonstrating all major features of the
[flutter_wacom_signature_pad](https://pub.dev/packages/flutter_wacom_signature_pad) package.

## Running the example

```bash
cd example
flutter pub get
flutter run -d windows
```

> **Windows only.** The Wacom STU SDK is a Windows native library.

---

## What's inside

| Screen | Description |
|---|---|
| **WacomPdfSigner** | Drop-in widget — downloads a PDF, lets the user place signature boxes, captures pen input from the Wacom tablet, then uploads the signed PDF |
| **HomeScreen** | Built-in home screen with a recent-documents list and Wacom connection toggle |
| **Connection Management** | Manually control connect/disconnect via `wacomConnectionProvider`; embed `WacomConnectButton` in your own app bar |
| **SignatureDialog** | Open the signature capture dialog standalone and receive the signature image |
| **Self-Signed SSL** | `trustSelfSignedCertificate: true` for internal corporate servers |

---

## Key files

| File | Purpose |
|---|---|
| `lib/main.dart` | Menu screen + all five demo screens |
| `pubspec.yaml` | Declares `flutter_wacom_signature_pad` as a path dependency |
| `windows/` | Windows runner generated by Flutter |

---

## Hardware requirement

A [Wacom STU](https://www.wacom.com/en-us/products/pen-displays/wacom-stu) signature pad
connected over USB. If the tablet is not connected, `SignatureDialog` falls back to
mouse/touch input on-screen.
